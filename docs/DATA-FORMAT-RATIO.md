# 持股比例数据格式说明

## 当前支持的格式

系统当前支持以下持股比例格式：

### 1. 百分比数值格式（推荐）✅

**格式**: 直接使用数字表示百分比

**示例**:
- `1` 表示 1%
- `60` 表示 60%
- `100` 表示 100%
- `17.09` 表示 17.09%

**Excel 中填写**:
```
| 持股比例 |
|---------|
| 1       |
| 60      |
| 100     |
| 17.09   |
```

**显示结果**:
- 1.00%
- 60.00%
- 100.00%
- 17.09%

### 2. 带百分号的字符串格式 ✅

**格式**: 数字 + %

**示例**:
- `1%` 表示 1%
- `60%` 表示 60%
- `100%` 表示 100%

**Excel 中填写**:
```
| 持股比例 |
|---------|
| 1%      |
| 60%     |
| 100%    |
```

**显示结果**:
- 1%
- 60%
- 100%

### 3. 小数格式（已禁用）❌

**格式**: 0-1 之间的小数

**示例**:
- `0.01` 表示 1%
- `0.6` 表示 60%
- `1` 表示 100%

**状态**: 已注释掉，因为与百分比数值格式冲突

**问题**: 
- `1` 会被识别为 100%（小数格式）而不是 1%（百分比数值格式）
- 造成数据混淆

## 为什么禁用小数格式？

### 问题场景

用户的数据使用百分比数值格式：

```excel
| 公司名称 | 股东名称 | 持股比例 |
|---------|---------|---------|
| 京海控股 | 张三    | 1       |  ← 期望显示 1%
| 京海控股 | 李四    | 60      |  ← 期望显示 60%
```

但是系统同时支持小数格式，导致：
- `1` 被识别为小数 1.0，显示为 100% ❌
- `60` 被识别为百分比数值，显示为 60% ✅

### 解决方案

注释掉小数格式的支持，统一使用百分比数值格式：

```javascript
// 注释前（支持三种格式）
static formatRatio(ratio) {
  // ...
  if (numRatio <= 1) {
    return `${(numRatio * 100).toFixed(2)}%`  // 小数格式
  }
  return `${numRatio.toFixed(2)}%`  // 百分比数值格式
}

// 注释后（只支持两种格式）
static formatRatio(ratio) {
  // ...
  // if (numRatio <= 1) {
  //   return `${(numRatio * 100).toFixed(2)}%`  // 已禁用
  // }
  return `${numRatio.toFixed(2)}%`  // 百分比数值格式
}
```

## 如何填写 Excel

### 推荐方式（百分比数值）

直接填写数字，不带百分号：

```excel
| 公司名称 | 股东名称 | 持股比例 |
|---------|---------|---------|
| 京海控股 | 张三    | 60      |
| 京海控股 | 李四    | 40      |
| 京海科技 | 京海控股 | 100     |
| 武汉控股 | 李四    | 17.09   |
```

### 也可以（带百分号）

填写数字 + %：

```excel
| 公司名称 | 股东名称 | 持股比例 |
|---------|---------|---------|
| 京海控股 | 张三    | 60%     |
| 京海控股 | 李四    | 40%     |
| 京海科技 | 京海控股 | 100%    |
```

### 不要使用（小数格式）

❌ 不要使用 0-1 之间的小数：

```excel
| 公司名称 | 股东名称 | 持股比例 |
|---------|---------|---------|
| 京海控股 | 张三    | 0.6     | ❌ 会显示为 0.60%
| 京海控股 | 李四    | 0.4     | ❌ 会显示为 0.40%
```

## 代码位置

持股比例格式化代码位于：

**文件**: `src/data/adapters/apiDataAdapter.js`

**函数**: `formatRatio(ratio)`

```javascript
static formatRatio(ratio) {
  if (!ratio) {
    return '0.00%'
  }

  // 如果已经包含 %，直接返回
  if (typeof ratio === 'string' && ratio.includes('%')) {
    return ratio
  }

  // 转换为数字并格式化
  const numRatio = parseFloat(ratio)
  
  if (isNaN(numRatio)) {
    return '0.00%'
  }

  // 注释掉小数格式支持（0-1），因为用户数据是百分比数值格式
  // 如果需要支持小数格式，取消下面的注释
  // if (numRatio <= 1) {
  //   return `${(numRatio * 100).toFixed(2)}%`
  // }

  // 百分比数值格式（0-100）
  // 例如：1 表示 1%，60 表示 60%，100 表示 100%
  return `${numRatio.toFixed(2)}%`
}
```

## 如何恢复小数格式支持？

如果将来需要支持小数格式，取消注释即可：

```javascript
static formatRatio(ratio) {
  // ...
  
  // 取消注释这部分
  if (numRatio <= 1) {
    return `${(numRatio * 100).toFixed(2)}%`
  }
  
  return `${numRatio.toFixed(2)}%`
}
```

但是要注意：
- 这会导致 `1` 被识别为 100%
- 需要确保数据中没有 1% 的情况
- 或者使用 `1%` 字符串格式来表示 1%

## 测试用例

### 测试数据

```javascript
formatRatio(1)      // 1.00%  ✅
formatRatio(60)     // 60.00% ✅
formatRatio(100)    // 100.00% ✅
formatRatio(17.09)  // 17.09% ✅
formatRatio('1%')   // 1%     ✅
formatRatio('60%')  // 60%    ✅
formatRatio(0.6)    // 0.60%  ⚠️ 注意：不是 60%
formatRatio(0)      // 0.00%  ✅
formatRatio(null)   // 0.00%  ✅
formatRatio('')     // 0.00%  ✅
```

### 预期结果

| 输入 | 输出 | 说明 |
|------|------|------|
| 1 | 1.00% | 百分比数值 |
| 60 | 60.00% | 百分比数值 |
| 100 | 100.00% | 百分比数值 |
| 17.09 | 17.09% | 百分比数值 |
| '1%' | 1% | 字符串格式 |
| '60%' | 60% | 字符串格式 |
| 0.6 | 0.60% | ⚠️ 小数格式已禁用 |
